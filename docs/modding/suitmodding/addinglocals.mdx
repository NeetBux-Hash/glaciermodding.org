## Pt. 1: **LOCATE YOUR LOCR** 
*A* <Tooltip text={<><strong>LOCR File</strong><span className="icon-alert"></span></>} tooltip="A file holding the game’s text, like names and descriptions, organized for different languages." /> *is basically a "container" for the game's text you see in game.* 


### 1. Identify your LOCR
Almost all <Tooltip text={<strong>localisation</strong>} tooltip="The process of adapting content for different languages and regions, such as translating text." /> (text you see on screen) are inside <Tooltip text={<strong>LOCR Files</strong>} tooltip="A file holding the game’s text, like names and descriptions, organized for different languages." />. Below are some of their <Tooltip text={<><strong>HASH</strong><span className="icon-alert"></span></>} tooltip="A unique identifier that acts as the address or coordinates for a file." /> lines: 
- Season 1 suits: `00985A1100E5EDDC`,
- Season 2 suits: `009F430D046716BE` ,
- Season 3 suits: `004B8C5124A49543` 

<details>
<summary>about **HASH**</summary>
:::info HASH
You can interpret <Tooltip text={<strong>HASH</strong>} tooltip="A unique identifier that acts as the address or coordinates for a file." /> lines as a file's specific address or coordinates. They are the file's *unique identifiers* and can bring us straight to the file.
>You will constantly have to use these throughout modding.
:::
</details>
[[Return to Adding More LOCR files]](#adding-more-locr-files)
### 2. Go to your LOCR  
- Go to the <Tooltip text={<strong>game contents</strong>} tooltip="A place in Glacierkit" /> tab on the left
- copy and paste your relevant <Tooltip text={<strong>HASH</strong>} tooltip="A unique identifier that acts as the address or coordinates for a file." /> on the search bar and hit enter. 

You'll see a map tree or *file path* leading to a <Tooltip text={<strong>LOCR File</strong>} tooltip="A file holding the game’s text, like names and descriptions, organized for different languages." />.
:::tip LOCR FILE

A <Tooltip text={<strong>LOCR File</strong>} tooltip="A file holding the game’s text, like names and descriptions, organized for different languages." /> can be identified with 2 attributes:
    - named like `randomwords.sweetmenutext].pc_localized-textlist` 
    - having a language or translation icon.
:::
- <Tooltip text={<strong>Open it</strong>} tooltip="By clicking it" />


### 3. Confirm your LOCR
You will see a *Preview* text box with code on the right and <Tooltip text={<strong>additional info on top</strong>} tooltip="The file's properties are displayed on top after clicking it" /> about the <Tooltip text={<strong>LOCR File</strong>} tooltip="A file holding the game’s text, like names and descriptions, organized for different languages." /> you opened.

Confirm you have the right file by checking:
- `LOCR` type of file 
- `HASH` code.

If you want to locate any *other text in-game*, you can go to the [**advanced search section**](#advanced-search-guide).

## Pt. 2: **GET YOUR STRING**
*Every line of code in the* <Tooltip text={<strong>LOCR File</strong>} tooltip="A file holding the game’s text, like names and descriptions, organized for different languages." /> *consists of a* <Tooltip text={<><strong>string</strong><span className="icon-alert"></span></>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." /> *attached to the actual text you see in-game*. 

Lets break a line down to understand the parts better.

 Example of a suit's name:
 ```json
 "989928F2-06D6-42F3-871A-353F07DEF969_LEGACY_HERO_BLOODMONEYSUIT_M_HPA2293_NAME_": "Blood Money Suit"
 ```
    - The <Tooltip text={<strong>string</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." /> is: `989928F2-06D6-42F3-871A-353F07DEF969_LEGACY_HERO_BLOODMONEYSUIT_M_HPA2293_NAME_`
    - the text you see in-game is: `Blood Money Suit`

### 1. Find your Suit
To find the suit name you want to modify easily instead of having to scroll, you will have to enter search mode. To do that: 
- Click inside the text box of code and press `Ctrl + F` to open the search function.
- type in your suit's name 
:::warning Search Mode
If you don’t click inside the text box first, the search won’t work properly and may look for text elsewhere in  <Tooltip text={<strong>GlacierKit</strong>} tooltip="A modding tool used for editing and managing game files in Hitman: World of Assassination." />.
>This feature is useful for other situations but doesn’t apply here.
:::
[[Return to Advanced Search]](#6-focus-on-the-right-file)
 
### 2. Copy the string
*You will only need the* <Tooltip text={<strong>string</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." /> *for the next part so copy it without the quotation marks!*

Like so:
`989928F2-06D6-42F3-871A-353F07DEF969_LEGACY_HERO_BLOODMONEYSUIT_M_HPA2293_NAME_`
<div id="multilangua"></div>

You will see your <Tooltip text={<strong>string</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." /> attached to multiple languages of your suit's name. 
This shows that a specific <Tooltip text={<strong>string</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." /> will always be the same in every other language `{array}` of a <Tooltip text={<strong>LOCR File</strong>} tooltip="A file holding the game’s text, like names and descriptions, organized for different languages." />.

## Pt. 3: **CONVERT TO DECIMAL**
We need to convert the <Tooltip text={<strong>string</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." /> to a <Tooltip text={<><strong>decimal number</strong><span className="icon-alert"></span></>} tooltip="A numeric representation derived from a string, used for reference in the manifest file." /> so we can use it in our <Tooltip text={<strong>manifest file</strong>} tooltip="A file in your mod folder that holds important information and can execute code" />.
That's what the <Tooltip text={<><strong>Text Tools tab</strong><span className="icon-alert"></span></>} tooltip="A tab in GlacierKit that provides tools for converting text and hashes for game modifications." /> is for.
>Converting the <Tooltip text={<strong>string</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." /> to a <Tooltip text={<strong>decimal</strong>} tooltip="A numeric representation derived from a string, used for reference in the manifest file." /> allows us to use it as a *"key"* in the localisation override manifest.


### 1. Go to Text Tools 
- Go to the <Tooltip text={<strong>Text Tools</strong>} tooltip="A tab in GlacierKit that provides tools for converting text and hashes for game modifications." /> tab
- Locate the <Tooltip text={<><strong>Localisation Hash Calculator</strong><span className="icon-alert"></span></>} tooltip= "A tool that converts your string into a format the manifest can use, generating both hex and decimal codes." /> box. 

### 2. Convert your string
- Paste your <Tooltip text={<strong>string</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." /> inside that box
- Copy *only* the converted <Tooltip text={<strong>decimal number</strong>} tooltip="A numeric representation derived from a string, used for reference in the manifest file." />. 

>*We now have everything needed for the next part:*
 >- a <Tooltip text={<strong>decimal form</strong>} tooltip="A numeric representation derived from a string, used for reference in the manifest file." /> of our <Tooltip text={<strong>string</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." />
 >- the <Tooltip text={<strong>HASH</strong>} tooltip="A unique identifier that acts as the address or coordinates for a file." /> from the <Tooltip text={<strong>LOCR File</strong>} tooltip="A file holding the game’s text, like names and descriptions, organized for different languages." />

:::tip windows clipboard
Easily manage multiple copied items using <Tooltip text={<strong>Windows clipboard</strong>} tooltip="A built-in tool that stores multiple copied items." />. Press `Windows + V` to activate it. 

Now copy both your: 
- LOCR's <Tooltip text={<strong>HASH</strong>} tooltip="A unique identifier that acts as the address or coordinates for a file." /> and 
- <Tooltip text={<strong>decimal</strong>} tooltip="A numeric representation derived from a string, used for reference in the manifest file." />.
>Make it a habit to paste using `Windows + V` instead of `Ctrl + V` from now on. 

 *It might seem overkill for 2 lines of codes but it'll pay off and it beats* *<Tooltip text={<italic>using notepad</italic>} tooltip="💀" />*  
 
:::

## Pt. 4: **UPDATE MANIFEST**
*Here we will be adding a* <Tooltip text={<><strong>localisation override code</strong><span className="icon-alert"></span></>} tooltip="A way to replace or modify in-game text, like names or descriptions, with your own custom text. It allows you to customize how text appears without altering the core game files." /> *line to your* <Tooltip text={<strong>manifest file</strong>} tooltip="A file in your mod folder that holds important information" />. *It will contain:*
- <Tooltip text={<strong>HASH</strong>} tooltip="A unique identifier that acts as the address or coordinates for a file." /> of your <Tooltip text={<strong>LOCR File</strong>} tooltip="A file holding the game’s text, like names and descriptions, organized for different languages." />
- <Tooltip text={<strong>decimal form</strong>} tooltip="A numeric representation derived from a string, used for reference in the manifest file." /> of your <Tooltip text={<strong>string</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." />
- *Your new text*

### 1. Add the localisation override line 

- Open your <Tooltip text={<strong>manifest file</strong>} tooltip="A file in your mod folder that holds important information and can execute code" />
- Add this line: `"localisationOverrides": {}`

<details>

<summary>How-to Manifest</summary>

Theres *2 methods* to doing this:
- *Method 1*: 
    - just copying: `"localisationOverrides": {}`
    - pasting it in your <Tooltip text={<strong>manifest file</strong>} tooltip="A file in your mod folder that holds important information and can execute code" />

 Or

- *Method 2*:
    - manually start typing it in, starting with `"` and then continue typing
    - navigate the *suggestions menu* to find `localisationOverrides`.
    

>Either method should result in:

   ```json
   "localisationOverrides": {}
   ```
:::tip using "suggestions"
*Now you know how to use the* <Tooltip text={<strong>suggestions feauture</strong>} tooltip="This feauture is only supported in Glacierkit or in VS code with a schema." />. It will auto-complete most of the tedious functions in the following steps like:
- adding "quotes", 
- adding one of these `:`
- adding brackets
- and more

>Suggestions will show up as soon as you type a **quote** `"`
:::
</details>
### 2. Insert LOCR HASH 
- Place your <Tooltip text={<strong>LOCR HASH</strong>} tooltip="the code that appeared on top when you clicked the file" /> inside the curly brackets in "quotes" and add another `:` and `{}` after that

Result:

   ```json
   "localisationOverrides": {
       "yourLOCRhash": {}
   }
    ```

### 3. Set your language

- Add `"english": {}` to indicate which language to override.
<div id="syntax"></div>
   Result:
   ```json
   "localisationOverrides": {
       "yourLOCRhash": {
           "english": {}
       }
   }
   ```


### 4. Add your decimal and text
- Inside the language brackets, paste your <Tooltip text={<strong>decimal number</strong>} tooltip="A numeric representation derived from a string, used for reference in the manifest file." /> in "quotes"
- Add your new text... Also in "quotes".

Result:
```json
"localisationOverrides": {
    "yourLOCRhash": {
        "english": {
            "yourconvertednamedecimal": "New Suit Name"
        }
    }
}
````

<details>
<summary>**Syntax**</summary>
:::info Syntax
If you're paying attention, you might notice that we're setting up the <Tooltip text={<strong>localisation override code</strong>} tooltip="A way to replace or modify in-game text, like names or descriptions, with your own custom text. It allows you to customize how text appears without altering the core game files." /> in the <Tooltip text={<strong>manifest file</strong>} tooltip="A file in your mod folder that holds important information and can execute code" /> 
just like our <Tooltip text={<strong>LOCR File</strong>} tooltip="A file holding the game’s text, like names and descriptions, organized for different languages." /> 
where we got the <Tooltip text={<strong>strings</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." /> from. 
Both will be using the same <Tooltip text={<strong>'Syntax'</strong>} tooltip="A set of rules that define how code is written and organized." />.


> *Even better too if you have noticed how similar this method is to [the previous *blobs article*](addingblobs.md#step-3-set-up-a-blobsfolder) where we 'recreated'* <Tooltip text={<strong>an assembly path</strong>} tooltip="Big thumb if you connected that HASH lines are 'coded' assembly paths = filePaths + fileNames." />
:::
</details>

### 5. (Optionally) Add more languages
#### Same Strings:
*In [Part 2](#2-copy-the-string) of the tutorial we demonstrated that* <Tooltip text={<strong>strings</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." /> *are the same in each* <Tooltip text={<strong>language array</strong>} tooltip="A single language array with multiple entries of strings each attached to text you see in-game" />
>*This will make adding languages for each entry very easy.*
    

To support more languages, simply:
- make another <Tooltip text={<strong>language bracket</strong>} tooltip="A single language array that can contain multiple entries of strings each attached to text you see in-game" /> 
- add the <Tooltip text={<strong>same decimal</strong>} tooltip="A numeric representation derived from a string, used for reference in the manifest file." /> with your text


Example of a new language line:
<div id="langcode"></div>
```json
"localisationOverrides": {
        "yourLocrHASH": {
            "english": {
                "yourconverteddecimal": "New Suit Name"
            },
            "french": {
                "yourconverteddecimal": "Niveua Costume nome"
            }
        }
    }
```
<details>
<summary>Language Codes</summary>

:::info language codes
- The reason why `french` is used here and not `français` is because the <Tooltip text={<strong>manifest's</strong>} tooltip="A file in your mod folder that holds important information and can execute code" /> <Tooltip text={<strong>schema</strong>} tooltip="A structure used to validate and provide auto-completion for your manifest, helping you write error-free code." /> is coded in English. 
- As for why it doesn't use the same <Tooltip text={<strong>language codes</strong>} tooltip="The language identifiers in each language array of a LOCR file" /> as the actual <Tooltip text={<strong>LOCR Files</strong>} tooltip="A file holding the game’s text, like names and descriptions, organized for different languages." /> like `en` or `fr`, is because the <Tooltip text={<strong>manifest's</strong>} tooltip="A file in your mod folder that holds important information and can execute code" /> <Tooltip text={<strong>schema</strong>} tooltip="A structure used to validate and provide auto-completion for your manifest, helping you write error-free code." /> is coded like that. 

To see how all languages have to be written down in the <Tooltip text={<strong>manifest</strong>} tooltip="A file in your mod folder that holds important information and can execute code" />, see the <Tooltip text={<strong>manifest info in SMF</strong>} tooltip="It's the book icon on the left after enabling developer mode." />. 
Or *[click here](#all-languages)*
:::
</details>
### 6. Add the description
*Now try the same for the description as well.*

To add *another* <Tooltip text={<strong>string</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." /> from the same <Tooltip text={<strong>LOCR File</strong>} tooltip="A file holding the game’s text, like names and descriptions, organized for different languages." /> :

- convert the <Tooltip text={<strong>string</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." /> from the <Tooltip text={<strong>suit's description</strong>} tooltip="Usually right below the string of your suit's name" /> in <Tooltip text={<strong>Text Tools</strong>} tooltip="A tab in GlacierKit that provides tools for converting text and hashes for game modifications." />
- add your new <Tooltip text={<strong>converted decimal</strong>} tooltip="A numeric representation derived from a string, used for reference in the manifest file." /> in the <Tooltip text={<strong>manifest</strong>} tooltip="A file in your mod folder that holds important information and can execute code" /> with the new text 
- *(Optionally)* add another language for the new description as well

Example:
```json
"localisationOverrides": {
        "yourLocrHASH": {
            "english": {
                "yourconverteddecimal": "New Suit Name",
                "yourotherconverteddecimal": "New Suit Description"
            },
            "french": {
                "yourconverteddecimal": "Niveua Costume nome",
                "yourotherconverteddecimal": "Description du nouveau costume"
            }
        }
    }
```
>*This recreates how the* <Tooltip text={<strong>LOCR File</strong>} tooltip="A file holding the game’s text, like names and descriptions, organized for different languages." /> *is internally set up by using the same* <Tooltip text={<strong>syntax</strong>} tooltip="A set of rules that define how code is written and organized." />.

## More on LocalisationOverrides

### Adding More LOCR Files

If you need to modify another <Tooltip text={<strong>LOCR File</strong>} tooltip="A file holding the game’s text, like names and descriptions, organized for different languages." />: 

1. **Insert a Comma**: After your previous <Tooltip text={<strong>LOCR Array</strong>} tooltip="an array in the manifest resembling the game's LOCR file syntax" />  in the <Tooltip text={<strong>manifest</strong>} tooltip="A file in your mod folder that holds important information and can execute code" />.
2. **Repeat [the steps from start](#1-identify-your-locr)** (or continue here)
2. **Add a New** <Tooltip text={<strong>LOCR Array</strong>} tooltip="an array in the manifest resembling the game's LOCR file syntax" />: Use the same structure, making sure to follow the <Tooltip text={<strong>syntax rules</strong>} tooltip="A set of rules that define how code is written and organized." />.
3. <Tooltip text={<strong>Repeat the Conversion</strong>} tooltip="Using the Localisation Hash calculator" />: Use <Tooltip text={<strong>Text Tools</strong>} tooltip="A tab in GlacierKit that provides tools for converting text and hashes for game modifications." /> again to convert any new <Tooltip text={<strong>strings</strong>} tooltip= "An identifier or 'key' used by LOCR files to display unique pieces of text, like names or descriptions." /> into <Tooltip text={<strong>decimals</strong>} tooltip="A numeric representation derived from a string, used for reference in the manifest file." />.
4. **Update the** <Tooltip text={<strong>manifest</strong>} tooltip="A file in your mod folder that holds important information and can execute code" />: Add the new language entries, keeping everything consistent.

Example of every aspect from this tutorial applied:
```json
"localisationOverrides": {
    "yourLocrHASH": {
        "english": {
            "yourconverteddecimal": "New Suit Name",
            "yourotherconverteddecimal": "New Suit Description/or Other New Suit name"
        }
    },
    "otherLocrHASH": {
        "english": {
            "somenewconverteddecimal": "More New Suit Name",
            "someotherconverteddecimal": "More New Suit Description/or Other More New Suit name"
        },
        "spanish": {
            "somenewconverteddecimal": "Más nombre de traje nuevo",
            "someotherconverteddecimal": "Más descripción del traje nuevo/u otro nombre del traje más nuevo"
        }
    }
}
```

*Remember to keep your* <Tooltip text={<strong>manifest</strong>} tooltip="A file in your mod folder that holds important information and can execute code" /> *clean and organized to avoid errors.*

### Advanced Search Guide
[[Return to tutorial]](#3-confirm-your-locr)

When you need to find text for something other than suits, we use the **advanced search funtion**. 

Start with:

1. Click on the <Tooltip text={<strong>Advanced Search Tab</strong>} tooltip="A tab in GlacierKit for performing extended searches that go beyond the 'game contents tab' search abilities. It has a magnifying glass icon and, is like all other tabs, on the left side." />

2. **Set Your Search Parameters**:
   - Check only the `chunk0` box.
    - Tick the **"localisation"** option.

    :::note
        - *Chunk0 holds most of the text data, making the search more efficient.*

        - The latter will refine your search to only localisation-related text files.
    

    :::


3. **Try Multiple Case Variations**:

*Advanced search is* ***case sensitive*** *try multiple type of search operations at the* ***same time***

    Search using different case-sensitive variations of your text. Examples:
     - "Blood Money"
     - "blood money"
     - "BLOOD MONEY"
     - "bloodmoney"

:::tip advanced multiple search   

You can continue to input new searches while others are still in progress.

### Glossary

This is the Glossary containing the most important keywords
- [**localisation override**](#pt-4-update-manifest): A way to replace or modify in-game text, like names or descriptions, with your own custom text. It allows you to customize how text appears without altering the core game files.

- [**LOCR File**](#1-identify-your-locr): A file holding the game’s text, like names and descriptions, organized for different languages.

- [**HASH**](#1-identify-your-locr): A unique identifier that acts as the address or coordinates for a file. Basically an assembly path thats coded

- [**String**](#pt-2-get-your-string): An identifier or 'key' used by LOCR files to attached to unique pieces of text, like names or descriptions.

- [**Text Tools**](#pt-3-convert-to-decimal): A tab in GlacierKit that provides tools for converting text and hashes for game modifications.

- [**Localisation Hash Calculator**](#pt-3-convert-to-decimal): A tool that converts your string into a format the manifest can use, generating both hex and decimal codes.

- [**Convert**](#2-convert-your-string): Using Text Tool's Localisation Hash Calculator to convert a string to its hex or decimal form.

- [**Decimal forn/number**](#2-convert-your-string): A numeric representation derived from a string, used for reference in the manifest file.

- [**manifest file**](#pt-4-update-manifest): A file in your mod folder that holds important information and can execute code.

- [**Syntax**](#syntax): A set of rules that define how code is written and organized.

- [**language array**](#5-optionally-add-more-languages): A single language array with multiple entries of strings each attached to text you see in-game.

- [**language codes**](#langcode): The language identifiers in each language array of a LOCR file.

   ```json
           }
       }
   }
   ```
